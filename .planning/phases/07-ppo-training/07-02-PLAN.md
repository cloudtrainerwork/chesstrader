# Phase 7 Plan 2: Curriculum Learning Framework

## Goal
Design and implement a progressive curriculum learning system that gradually increases training difficulty through market regime complexity, strategy sophistication, and risk parameters to ensure stable and effective PPO training for options trading.

## Dependencies
- **Phase 7-01 Complete**: PPO algorithm implementation
- **Phase 6 Complete**: OptionsTrainingEnvironment with MarketRegime support
- **Existing Infrastructure**: Strategy types, market regimes from environments

## Tasks

### Task 1: Design Progressive Difficulty Levels
**File**: `src/training/curriculum/levels.py`

Create structured difficulty progression system:
- **Level 1**: Simple market conditions (low volatility, predictable trends)
- **Level 2**: Moderate market complexity (normal volatility, some regime changes)
- **Level 3**: Complex market conditions (high volatility, frequent regime switches)
- **Level 4**: Extreme market scenarios (black swan events, market crashes)

Define progression criteria and difficulty metrics:
- Market volatility thresholds
- Position complexity parameters
- Episode length variations
- Risk parameter adjustments

**Key Components**:
- `DifficultyLevel` enum with progression stages
- `CurriculumParameters` dataclass for level configuration
- Transition criteria between levels
- Performance thresholds for advancement

### Task 2: Implement Adaptive Curriculum Scheduler
**File**: `src/training/curriculum/scheduler.py`

Create intelligent curriculum progression system:
- Performance-based level advancement
- Automatic difficulty adjustment based on success rate
- Fallback mechanisms for struggling performance
- Multi-objective progression criteria (profit, risk-adjusted returns, Sharpe ratio)

**Advancement Logic**:
- Success rate > 70% for 10 consecutive episodes → advance level
- Success rate < 30% for 20 consecutive episodes → reduce level
- Plateau detection and difficulty boost mechanisms
- Manual override capabilities for testing

**Key Methods**:
- `should_advance_level()`: Check advancement criteria
- `should_reduce_level()`: Check fallback criteria
- `get_current_parameters()`: Get current curriculum configuration
- `update_performance()`: Track performance metrics

### Task 3: Create Strategy-Specific Curricula
**File**: `src/training/curriculum/strategies.py`

Implement specialized curriculum paths for different option strategies:
- **Iron Condor**: Start with wide spreads, reduce width over time
- **Iron Butterfly**: Begin with ATM positions, introduce OTM variations
- **Straddle**: Start with high volatility periods, add low volatility challenges
- **Strangle**: Progress from wide to narrow strike differences

**Strategy Progression**:
- Strike selection complexity (ATM → OTM → mixed)
- Expiration timing (long-term → short-term → mixed)
- Market regime exposure (favorable → neutral → adverse)
- Position sizing and risk management complexity

**Key Components**:
- `StrategyCurriculum` base class
- Strategy-specific curriculum implementations
- Adaptive strike selection algorithms
- Risk-adjusted progression metrics

### Task 4: Build Adaptive Difficulty Adjustment
**File**: `src/training/curriculum/adaptation.py`

Create dynamic curriculum system that adapts to agent performance:
- Real-time difficulty adjustment based on performance trends
- Multi-metric evaluation (P&L, drawdown, Sharpe ratio, win rate)
- Confidence intervals for performance assessment
- Plateau detection and curriculum intervention

**Adaptive Features**:
- Performance trend analysis (sliding window statistics)
- Automatic parameter tuning within difficulty levels
- Emergency fallback for persistent poor performance
- Progressive challenge injection for advanced agents

**Key Methods**:
- `analyze_performance_trend()`: Detect performance patterns
- `adjust_difficulty_parameters()`: Fine-tune current level
- `detect_plateau()`: Identify learning plateaus
- `recommend_intervention()`: Suggest curriculum changes

## File Specifications

### src/training/curriculum/levels.py
```python
class DifficultyLevel(Enum):
    BEGINNER = 1
    INTERMEDIATE = 2
    ADVANCED = 3
    EXPERT = 4

@dataclass
class CurriculumParameters:
    market_volatility: Tuple[float, float]  # (min, max) volatility range
    regime_stability: float  # 0-1, higher = more stable
    position_complexity: int  # 1-4, strategy complexity level
    episode_length_range: Tuple[int, int]  # (min, max) episode steps
    risk_multiplier: float  # Risk scaling factor

class CurriculumLevel:
    def __init__(self, level: DifficultyLevel)
    def get_parameters(self) -> CurriculumParameters
    def meets_advancement_criteria(self, performance_history: List[float]) -> bool
```

### src/training/curriculum/strategies.py
```python
class StrategyCurriculum(ABC):
    @abstractmethod
    def get_strike_selection_rules(self, level: DifficultyLevel) -> Dict[str, Any]
    @abstractmethod
    def get_expiration_rules(self, level: DifficultyLevel) -> Dict[str, Any]

class IronCondorCurriculum(StrategyCurriculum):
    def get_strike_selection_rules(self, level: DifficultyLevel) -> Dict[str, Any]
    def get_position_sizing_rules(self, level: DifficultyLevel) -> Dict[str, Any]

class CurriculumFactory:
    @staticmethod
    def create_curriculum(strategy_type: StrategyType) -> StrategyCurriculum
```
